# Restaurant Orders Analysis 

## Overview

This project analyzes a quarter's worth of restaurant order data using **SQL** to uncover actionable business insights. The analysis focuses on menu performance, customer purchasing behavior, revenue generation, and ordering trends to help restaurant managers make data-driven decisions.

This project demonstrates the use of SQL for data analysis and business analysis, leveraging joins, Common Table Expressions (CTEs), window functions, aggregate functions, and self joins to answer real-world business questions and generate actionable insights.

---

## Dataset

**Source:** Maven Analytics – Restaurant Orders Dataset

[Dataset](https://mavenanalytics.io/data-playground/restaurant-orders)

### Dataset Description

The dataset contains restaurant order information from a fictional restaurant serving international cuisine.

It consists of two related tables:

### `menu_items`

Contains information about each menu item.

| Column | Description |
|--------|-------------|
| `menu_item_id` | Unique menu item identifier |
| `item_name` | Name of the menu item |
| `category` | Cuisine category |
| `price` | Item price |

### `order_details`

Contains individual items ordered by customers.

| Column | Description |
|--------|-------------|
| `order_details_id` | Unique record identifier |
| `order_id` | Order identifier |
| `order_date` | Date of the order |
| `order_time` | Time of the order |
| `item_id` | Ordered menu item |

## Project Objectives

The primary objective of this project is to answer business questions such as:

- How is the restaurant menu structured?
- Which menu items generate the highest revenue?
- Which cuisine categories contribute the most revenue?
- What are customers ordering most frequently?
- Which menu items are commonly purchased together?
- Which hours of the day generate the most orders and revenue?
- Which high-value orders contribute most to sales?

---

## SQL Concepts Demonstrated

This project applies several important SQL concepts, including:

- Aggregate Functions (`COUNT`, `SUM`, `AVG`)
- Joins (`INNER JOIN`, `LEFT JOIN`)
- Common Table Expressions (CTEs)
- Window Functions (`RANK()`, `OVER()`)
- Self Joins (Market Basket Analysis)
- CASE Statements
- Date & Time Functions

## Analysis Performed

### 1. Menu Analysis

- Total number of menu items
- Least and most expensive menu items
- Number of dishes in each category
- Average menu price by category
- Top 10 most expensive menu items

### 2. Order Analysis

- Restaurant operating date range
- Total number of orders
- Total items ordered
- Top 10 largest orders by number of items
- Average number of items per order
- Menu items that never received an order

### 3. Revenue & Sales Performance

- Revenue generated by each category
- Revenue contribution (%) by category
- Top 5 revenue-generating menu items
- Top 5 highest spending orders
- Detailed breakdown of the highest spending orders
- Revenue ranking of menu items within each category

### 4. Customer Purchasing Patterns

- Average order value
- Top 5 most frequently ordered menu items
- Market Basket Analysis (items frequently purchased together)

### 5. Time-Based Analysis

- Hours with the highest number of orders
- Hours generating the highest revenue

## Business Value

The analysis provides insights that can help restaurant managers:

- Identify top-performing and underperforming menu items.
- Understand which cuisine categories drive the most revenue.
- Recognize customer purchasing habits and popular item combinations.
- Optimize menu offerings based on customer demand.
- Schedule staff more efficiently during peak business hours.
- Improve upselling strategies using frequently purchased item combinations.
- Focus marketing efforts on high-value products and peak sales periods.
